# Law 0

Momentum OS існує, щоб будувати машини доходу. Машини доходу не існують, щоб будувати Momentum OS.

# Principles

1. Будуй системи, а не списки задач.
2. Захищай увагу: одна глибока дія важливіша за десять перемикань.
3. Шукай bottleneck, а не нову цікаву роботу.
4. Показуй лише наступний видимий крок.
5. Зберігай докази прогресу, а не настрій про прогрес.
6. Вимірюй правильну поведінку та факти, не зайнятість.
7. Нова ідея — не новий пріоритет без proof.
8. Якщо система додає бюрократію, її спрощують або видаляють.

# Known Enemies

- Нудьга → імпульсивна угода.
- Перфекціонізм → build до proof.
- Аналіз без дії → великий, нечіткий квест.
- Новизна → втеча від поточного bottleneck.
- Тільт → рішення в моменті замість правила.

# Daily Win

День виграно, якщо виконано одну корисну Mission або свідомо захищено капітал і увагу, коли правильною дією було не працювати чи не торгувати.
