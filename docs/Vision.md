# Purpose

Визначити, яку цінність має створити Momentum OS і якою повинна бути її перша робоча версія.

# Vision

Momentum OS — це не планер і не список задач. Це персональна система прийняття рішень, яка прибирає зайві варіанти та показує одну найціннішу дію для розвитку доходних машин.

# Mission

Допомогти Ярославу будувати системи доходу, а не просто бути зайнятим: працювати 4–5 глибоких годин на день і поступово зміщувати дохід від ручної роботи до активів та делегованих процесів.

# Product Promise

Після відкриття Momentum OS користувач за 10 секунд розуміє:

1. Яку одну дію робити наступні 30–90 хвилин.
2. Чому саме ця дія має найвищий важіль.
3. Яке вузьке місце конкретної машини вона усуває.
4. Що сьогодні свідомо не потрібно робити.

# Core Principles

1. **One Best Move.** Система рекомендує одну головну дію, а не список справ.
2. **Bottleneck First.** Місія існує лише тоді, коли прибирає вузьке місце однієї з машин.
3. **Fog of War.** Показуємо наступний видимий крок, а не весь шлях проєкту.
4. **Proof Before Build.** Нові машини спершу підтверджують попит, а потім отримують час на розробку й автоматизацію.
5. **Low Friction.** Оновлення займає до 90 секунд на день; система має бути легшою за хаос, який вона замінює.
6. **Behavior Over Busyness.** Важливі правильні рішення та завершені результати, а не кількість закритих задач.

# MVP

Перший реліз має вирішити один сценарій: користувач відкриває застосунок, бачить рекомендовану місію, працює над нею та коротко фіксує результат.

## Included

- Today: головна місія, коротке пояснення та максимум дві альтернативи.
- Income Engines: чотири машини, їхня стадія та поточне вузьке місце.
- Mission completion: результат, короткий доказ прогресу, наступний видимий крок.
- Weekly review: короткий огляд фактів і перегляд пріоритетів.

## Excluded

- Telegram-бот і мобільні застосунки.
- OpenAI API / AI COO.
- Повна RPG-гейміфікація, XP, рівні, досягнення й анімації.
- Автоматична оцінка фінансів, угод або поведінкових патернів.

# Success Metrics (First 30 Days)

1. У 80% робочих днів Ярослав відкриває Today перед першою глибокою сесією.
2. Відкриття Today дає ясну відповідь на «що робити зараз?» менш ніж за 10 секунд.
3. Щоденне оновлення займає до 90 секунд.
4. Щонайменше 20 завершених місій містять результат або доказ прогресу, а не лише позначку «зроблено».
5. Під час щотижневого огляду можна назвати принаймні одну реальну зміну в одній з машин.

# Non-Goals

- Замінити терапію, фінансового радника або ризик-менеджмент трейдингу.
- Зробити універсальний продукт для всіх користувачів з першої версії.
- Створити ще один великий проєкт, який відволікатиме від доходних машин.

# Definition of Done

- [x] Описано проблему, місію та обіцянку продукту.
- [x] Визначено принципи, межі MVP і критерії успіху на перші 30 днів.
- [x] Ясно відокремлено MVP від майбутніх ідей.
