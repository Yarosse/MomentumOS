# Purpose

Визначити, як Momentum OS представляє напрямок доходу — Engine. Engine потрібна не для списку проєктів, а щоб бачити його реальну стадію, вузьке місце та наступний важіль.

# Core Definition

**Engine** — це напрямок, який потенційно створює гроші, актив або систему зменшення ручної роботи. Кожна активна місія обов'язково належить одній Engine.

# Engine Fields (MVP)

| Field | Required | Description |
| --- | --- | --- |
| Name | Yes | Назва машини. |
| Type | Yes | `active_income`, `community`, `product`, `delegated_business`. |
| Status | Yes | `active`, `experiment`, `paused`, `retired`. |
| Stage | Yes | Поточна стадія розвитку; показуємо лише її, а не весь roadmap. |
| Bottleneck | Yes | Одна головна причина, що зараз обмежує рух машини. |
| Objective | Yes | Перевірюваний результат поточної стадії. |
| Last proof | No | Останній факт, який підтверджує прогрес. |
| Next visible step | No | Кандидат на наступну місію. |
| Updated at | Yes | Дата останньої зміни. |

# Status Rules

| Status | Meaning | Today behavior |
| --- | --- | --- |
| `active` | Машина має підтверджений потенціал або вже створює цінність. | Може отримувати головну місію. |
| `experiment` | Гіпотеза; попит або модель ще не доведені. | Отримує тільки місії для перевірки гіпотези. |
| `paused` | Свідомо зупинена, щоб не розпорошувати увагу. | Не пропонувати в Today. |
| `retired` | Закрита на підставі фактів. | Зберігати історію, але не пропонувати місії. |

# Stage Model

Стадії не є універсальною лінійною шкалою. Для MVP кожна Engine має коротку назву своєї поточної стадії та один критерій виходу з неї.

Приклади:

| Engine | Current stage | Exit criterion |
| --- | --- | --- |
| Scalping | «Зібрати статистику за правилами» | 20 угод, виконаних за чек-листом, із заповненим журналом. |
| Arbitrage community | «Зробити онбординг самостійним» | Новачок проходить шлях без ручної допомоги. |
| Telegram game | «Підтвердити попит» | Є розмови/тест, що підтверджують бажання цільових гравців користуватись продуктом. |
| Doors | «Підтвердити канал дилерів» | Щонайменше один потенційний дилер пройшов предметну розмову про співпрацю. |

# Bottleneck Rules

1. В активної Engine може бути тільки одне головне вузьке місце.
2. Вузьке місце повинне бути сформульоване як перевірювана перешкода, а не як абстракція. Наприклад, «немає статистики угод», а не «треба краще трейдити».
3. Якщо місія не зменшує поточне вузьке місце, вона не потрапляє в Today.
4. Вузьке місце переглядається на Weekly Review або після сильного нового факту.

# Initial Engines

## Scalping

- Type: `active_income`
- Status: `active`
- Stage: Зібрати статистику за правилами
- Bottleneck: немає перевіреної дисципліни та достатньої статистики угод
- Objective: 20 угод за чек-листом із зафіксованим результатом

## Arbitrage Community

- Type: `community`
- Status: `active`
- Stage: Зробити онбординг самостійним
- Bottleneck: нових людей не можна масштабувати без ручних пояснень
- Objective: один новачок проходить онбординг без допомоги

## Telegram Game

- Type: `product`
- Status: `experiment`
- Stage: Підтвердити попит
- Bottleneck: немає перевірки, що цільові гравці справді перейдуть на нову платформу
- Objective: отримати якісні інтерв'ю або тест попиту від потенційних гравців

## Doors

- Type: `delegated_business`
- Status: `experiment`
- Stage: Підтвердити канал дилерів
- Bottleneck: немає першого підтвердженого дилерського контакту
- Objective: провести одну предметну розмову з потенційним дилером

# Engine Detail Screen (Post-MVP)

В майбутньому екран Engine показуватиме лише:

1. Поточну стадію.
2. Вузьке місце.
3. Останній доказ.
4. Наступний видимий крок.
5. Історію змін стадій.

Фінансові графіки, великі беклоги, документація та всі далекі етапи не входять до цього екрана MVP.

# Acceptance Criteria

- [ ] У системі створено рівно чотири початкові Engine.
- [ ] Кожна Engine має статус, стадію, одне вузьке місце та критерій виходу.
- [ ] Місію неможливо створити без прив'язки до Engine.
- [ ] Paused і retired Engine не з'являються серед рекомендацій Today.
- [ ] На головному екрані показується тільки поточна стадія, а не повна карта шляху.
