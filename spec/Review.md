# Purpose

Перетворювати тижневий досвід на рішення, не витрачаючи понад 10 хвилин і не перетворюючи review на щоденник.

# Cadence

Раз на 7 днів. Якщо review пропущено, система не карає streak-ом: вона просто показує, що пріоритети застаріли.

# Five Questions

1. Які місії завершені й який у кожної доказ?
2. Яка Engine стала сильнішою та чому?
3. Яке вузьке місце не зрушилось?
4. Де був самообман або зайве перемикання?
5. Яка одна Engine має найбільший важіль наступного тижня?

# Output

- Один пріоритетний Engine.
- Одне актуальне bottleneck для кожної active Engine.
- Одне правило «не робити» на наступний тиждень.
- Одна recommended Mission для наступного Today.

# Acceptance Criteria

- [ ] Review займає до 10 хвилин.
- [ ] Усі відповіді спираються на факти завершених місій.
- [ ] Review оновлює щонайменше одну Engine або її пріоритет.
