# Purpose

Визначити головний екран MVP. Today має зняти питання «що робити зараз?» і запустити одну корисну сесію без довгого планування.

# User Problem

Коли перед Ярославом кілька машин та багато ідей, він витрачає енергію на аналіз варіантів або бачить увесь шлях і не починає. Звичайний список задач лише збільшує перевантаження.

# Product Decision

Today не показує беклог, усі етапи проєктів або докладну аналітику. На екрані є одна рекомендована місія й максимум дві альтернативи, що не суперечать поточному пріоритету.

# Screen Content

## 1. One Best Move

Обов'язковий головний блок.

| Field | Description |
| --- | --- |
| Mission title | Результат, який можна перевірити; не дія на кшталт «попрацювати». |
| Engine | Машина, яку рухає місія. |
| Bottleneck | Вузьке місце, яке усуває місія. |
| Duration | Оцінка від 30 до 90 хвилин. |
| Why now | Одне коротке речення про важіль цієї дії. |
| Start | Кнопка початку фокус-сесії. |

Приклад:

> **Перевірити, чи новачок проходить онбординг без моєї допомоги.**
>
> Машина: Arbitrage community · Вузьке місце: онбординг
>
> 45 хв · Бо це зменшує ручну підтримку й дозволяє масштабувати залучення людей.

## 2. Alternatives

До двох місій. Вони показуються лише коли головна місія не може бути виконана зараз через енергію, доступ до даних або зовнішню залежність.

Для кожної альтернативи показувати тільки назву, машину та тривалість. Не показувати повний список задач.

## 3. Today Boundary

Короткий блок «Сьогодні не робимо». До трьох пунктів, сформульованих конкретно.

Приклад:

- Не починати новий продукт.
- Не кодити Telegram-гру до перевірки попиту.
- Не відкривати угоду через нудьгу.

Це не покарання, а захист уваги від відомих патернів самообману.

## 4. Proof of Progress

Після завершення місії користувач бачить останній доказ руху: короткий факт, результат або посилання на артефакт. У MVP достатньо показувати один останній запис.

# Primary Flow

1. Користувач відкриває Today.
2. За 10 секунд читає головну місію, вузьке місце й причину.
3. Натискає Start або обирає одну альтернативу.
4. Після роботи натискає Complete.
5. Вводить один короткий результат: що стало фактом або що змінилося.
6. Система позначає місію завершеною та показує наступний видимий крок, але не весь шлях.

# Completion Rules

Місію можна завершити лише з результатом, що має один із типів:

- **Proof:** факт, цифра, відповідь користувача, заявка, угода або інший доказ.
- **Artifact:** посилання чи опис створеного гайда, шаблону, прототипу або документа.
- **Learning:** чітко сформульований висновок, який змінює наступне рішення.

Відповідь «попрацював» або «ще думаю» не завершує місію.

# Empty State

Якщо активної місії немає, Today показує тільки одну дію:

> «Визначити вузьке місце машини».

Користувач обирає одну з чотирьох машин і вводить коротку відповідь на питання: «Що зараз найбільше стримує її зростання?»

# Out of Scope

- Таймер Pomodoro, блокування сайтів, трекінг активності.
- Автоматичні рекомендації від AI.
- Графіки, рейтинги та складні метрики Momentum.
- Повний перегляд беклогу з Today.

# Acceptance Criteria

- [ ] Today відкривається одразу після входу в застосунок.
- [ ] На екрані є рівно одна головна місія.
- [ ] Головна місія має тривалість 30–90 хвилин, машину, вузьке місце та пояснення «чому зараз».
- [ ] Альтернативних місій не більше двох.
- [ ] Завершення місії вимагає одного короткого доказу результату.
- [ ] На Today немає довгого списку задач, беклогу чи майбутніх етапів.
