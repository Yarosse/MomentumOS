# Purpose

Перетворювати одне вузьке місце Engine на коротку, видиму й перевірювану дію без показу всього проєктного шляху.

# Definition

**Mission** — це одна сесія на 30–90 хвилин, прив'язана до однієї Engine і її поточного bottleneck. Вона описується через результат, а не через загальну активність.

# Required Fields

| Field | Description |
| --- | --- |
| Title | Перевірюваний результат: «Новачок проходить онбординг без допомоги». |
| Engine | Єдина Engine, яку рухає місія. |
| Bottleneck | Поточне вузьке місце Engine. |
| Why now | Одне речення про важіль місії. |
| Estimated minutes | Від 30 до 90 хвилин. |
| Status | `planned`, `active`, `completed`, `cancelled`. |
| Completion proof | Факт, артефакт або висновок після завершення. |

# Rules

1. Одночасно активна тільки одна Mission.
2. У Today є не більше двох planned alternatives.
3. Місії зі словами «створити продукт», «запустити бізнес», «покращити все» вважаються занадто великими й дробляться.
4. Якщо результат не вкладається у 90 хвилин, місія не готова.
5. Completion proof обов'язковий; «попрацював» не є результатом.
6. Нова ідея потрапляє в backlog, а не створює Mission без перегляду пріоритету.

# Mission Templates

## Validate

> Отримати відповідь на одну невідому бізнес-гіпотезу.

Приклад: «Провести три короткі розмови з гравцями в деберц і зафіксувати, чого їм бракує на наявних платформах».

## Build

> Створити малий артефакт, який зменшує ручну роботу або наближає перевірений продукт до запуску.

Приклад: «Створити онбординг-гайд, за яким один новачок виконує перший арбітражний крок без пояснень».

## Improve

> Покращити процес, що вже працює, на підставі факту.

Приклад: «Додати до чек-листа скальпінгу правило, яке закриває помилку з останніх п'яти угод».

# Completion Flow

1. Користувач натискає Complete.
2. Вводить 1–3 речення proof.
3. За потреби позначає: bottleneck зменшився / не змінився / змінився.
4. Система зберігає факт і пропонує наступний видимий крок, але не відкриває весь roadmap.

# Acceptance Criteria

- [ ] Місія не створюється без Engine і bottleneck.
- [ ] Оцінка часу — 30–90 хвилин.
- [ ] Одночасно активна одна місія.
- [ ] Завершення неможливе без proof.
- [ ] Today не показує далекі етапи після завершення.
